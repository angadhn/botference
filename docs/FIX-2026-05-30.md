# Botference reliability + UX fix ‚Äî 2026-05-30

Fixes a family of problems that all share one root cause: **the room transcript grows
unbounded (‚âà80% of it un-truncated tool output), and several code paths do O(N) or
O(N¬≤) work against a context-window estimate that can be wrong.** Symptoms: long chats
crash/wedge, the textbox lags while bots stream, reloads "stream in" slowly, and a
stuck relay leaves the session unusable. A correlated symptom ‚Äî trackpad "gibberish"
and selection-on-scroll ‚Äî comes from the same input loop stalling.

All changes are **source-only and backward-compatible**: existing session JSON loads
unchanged, no schema migration, new behavior applies to new turns only. They take
effect on the next app start (see `CUTOVER.md`).

## What changed, by thread

### Cross-cutting ‚Äî truncate tool output at capture
`core/cli_adapters.py` ‚Äî `_structured_output_blocks` cap lowered from **8000 ‚Üí 2000
chars** (`_TOOL_OUTPUT_BLOCK_LIMIT`) with a `[‚Ä¶ N more characters truncated]` marker.
Full output still lives in `raw_output` for the debug panes. This is the highest-leverage
change: a tool-heavy turn previously wrote 100s of KB into the transcript; that drove
the overflow crashes, the slow reloads, and the render lag.

### Thread A ‚Äî context measurement + relay/compaction
- **Bounded relay/late-join backfill** (`core/botference.py`): `context_after` and
  `context_since` now keep only the most-recent `_BACKFILL_MAX_CHARS` (60k) of history,
  with an elision marker. The handoff doc already carries the durable summary; the
  backfill only needs recent turns. *This is the fix for the wedge* ‚Äî previously a relay
  could rebuild a prompt as large as the session that triggered it, so recovery itself
  overflowed and the handoff got stuck in `pending_relay_handoffs`.
- **Overflow backstop** (`core/cli_adapters.py`): new `is_context_overflow()` matches
  provider overflow errors ("prompt is too long: X > Y", "exceeds the context window",
  `request_too_large`, ‚Ä¶). Both adapters set `AdapterResponse.context_overflow`.
- **Surface + recover** (`core/botference.py` `_update_pct`): on overflow, force maximum
  yield pressure (so a relay picks the mechanical tier) and emit an actionable
  `Run /relay @model` message. With the bounded backfill, that relay now succeeds.
- **Occupancy fallback fix** (`core/cli_adapters.py`): Claude's fallback occupancy now
  includes `cache_read_tokens` (the bulk of a resumed session), not just input +
  cache-creation.

### Thread B ‚Äî mouse input layer (ink-ui)
- `src/v2/terminalModes.ts`: `ENABLE_MOUSE_TRACKING` switched from `?1003h` (any-motion)
  to `?1002h` (button-event tracking). Any-motion turned every trackpad movement into a
  stream of escape sequences that leaked as "gibberish" when the loop stalled and fed
  phantom drag-selects. Scroll wheel is still reported.
- `src/v2/stdinFilter.ts`: horizontal-wheel buttons **66/67** are now consumed as scroll
  (previously misread as a *press* ‚Üí phantom selection on a two-finger swipe); and
  press/drag is only emitted for a **real held button** (bare motion, button bits == 3,
  is ignored).

### Thread C ‚Äî render / input latency (ink-ui)
- `src/layout.ts`: `preRenderLines` now uses a **per-entry FlatLine WeakMap cache**.
  Wrapping/markdown/syntax work is the dominant cost and during streaming/reload only one
  entry changes ‚Äî caching by entry identity makes streaming **O(1) per token** and reload
  **O(N)** instead of O(N¬≤).
- `src/App.tsx`: the transcript is wrapped in `useDeferredValue`, so keystrokes (urgent)
  aren't blocked behind a pane reflow while bots stream.

### Thread D ‚Äî reload bulk hydration
- `core/botference_ink_bridge.py` + `core/botference.py`: `_replay_restored_session` now
  sends history as a few batched `restore` events (`InkBridge.restore_entries`, 80/batch)
  instead of one emit + render + reflow per entry. Falls back to per-entry replay for any
  UIPort lacking `restore_entries`.
- `src/App.tsx`: new `restore` message case applies each batch with a single
  `appendEntries` state update.

### Thread E ‚Äî recovery script (not run against a live session)
`scripts/unwedge_session.py`: idempotent recovery for a wedged session ‚Äî clears stuck
`pending_relay_handoffs`, resets stale (>100) `yield_pressure`, optionally trims oversized
tool output. Two guards (a `pgrep -f` process check **and** a recent-mtime check) refuse to
run while botference is live, plus a timestamped `.bak` backup. Validated on a copy of the
real `ec53fd69` session: handoff cleared, pressures reset, all 369 transcript / 460
room_history entries intact.

## Decisions / deliberate scope

- **No change to `_CONTEXT_WINDOWS` (opus ‚Üí 1M).** The `‚Ä¶[1m]` model name already misses
  the table and falls back to 200k, and when the CLI genuinely reports a 1M window via
  `modelUsage` it is used. Robustness comes from the **bounded backfill + overflow
  backstop**, which are correct regardless of the window number ‚Äî rather than guessing it.
- **Relay stays user-triggered**, not auto. On overflow we surface an actionable `/relay`
  and raise pressure, instead of tearing down a session mid-turn behind the user's back.
- **`useDeferredValue` instead of extracting a memoized input component.** Lower risk in
  the ~1800-line `App.tsx`; combined with the per-entry cache it delivers the latency win.
  A full input-component split remains a possible follow-up.
- **Codex first-turn 0% occupancy** (until a cumulative baseline exists) is left as-is ‚Äî
  it is display-only and not a crash cause.
- **Tool-output cap = 2000 chars** trades some debug-pane fidelity for a ~4√ó smaller
  transcript; full output remains in `raw_output`.

## Risks / limitations
- Deferred transcript: panes may trail the input by a frame during heavy streaming (intended).
- Per-entry cache assumes entries are treated immutably (they are; the cache also compares
  `text` + `blocks` references defensively, so in-place mutation still invalidates).
- Bounded backfill means a relayed/late-joining model sees only recent history inline; older
  context lives in the handoff summary and is marked elided.
